# 状态字典

> **定义**: 工作流中所有状态的统一定义

---

## 工作流状态

### 全局状态

| 状态 | 触发者 | 含义 | 后续动作 |
|------|--------|------|----------|
| `[WORKFLOW_STARTED]` | 工作流入口 | 工作流启动 | 进入阶段 0 |
| `[WORKFLOW_COMPLETED]` | 阶段 4 通过 | 工作流完成 | 归档记录 |
| `[WORKFLOW_FAILED]` | 任一阶段失败 | 工作流失败 | 返回修复 |
| `[WORKFLOW_PAUSED]` | 用户请求 | 工作流暂停 | 等待恢复 |
| `[WORKFLOW_CANCELLED]` | 用户请求 | 工作流取消 | 清理资源 |

| 状态 | 触发者 | 含义 | 后续动作 |
|------|--------|------|----------|
| `[STAGE_0_STARTED]` | 工作流入口 | 阶段 0 开始 | 执行需求分析 |
| `[STAGE_0_EVALUATING]` | 复杂度评估 | 正在评估复杂度 | 等待评估完成 |
| `[STAGE_0_WAITING_CONFIRM]` | 评估完成 | 等待用户确认 | 用户确认后进入阶段 1 |
| `[STAGE_0_PASSED]` | 用户确认 | 阶段 0 通过 | 进入阶段 1 |
| `[STAGE_0_FAILED]` | 评估失败 | 阶段 0 失败 | 返回需求澄清 |
