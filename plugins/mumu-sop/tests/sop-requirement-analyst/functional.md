# sop-requirement-analyst 功能测试

## 测试 1: 基础需求分析

### 输入
```
用户需要一个用户注册功能，支持邮箱注册和手机号注册。
注册后需要发送验证邮件或短信。
```

### 预期输出
- 生成澄清记录文档 (specs/{name}-clarification.md)
- 包含 4 轮提问的完整记录
- 用户确认后生成规范文档 (specs/{name}-spec.md)
- 规范包含验收标准

### 验证点
- [ ] 是否进行了多轮提问
- [ ] 是否区分了长期目标与短期实现
- [ ] 是否生成了可测试的验收标准
- [ ] 规范文档是否符合 P0 级约束

---

## 测试 2: 复杂需求分析

### 输入
```
需要一个订单管理系统，支持：
- 创建订单
- 查询订单状态
- 取消订单
- 订单支付
- 订单退款
需要对接第三方支付系统，支持多种支付方式。
```

### 预期输出
- 识别多个功能模块
- 明确模块间依赖关系
- 生成 P1 系统级规范
- 定义接口契约

### 验证点
- [ ] 是否识别了功能边界
- [ ] 是否明确了第三方集成约束
- [ ] 是否定义了清晰的验收标准

做一个报表功能
