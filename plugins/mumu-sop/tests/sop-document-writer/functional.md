# 功能测试

## 测试 1：技术模式 - 创建 API 文档

### 输入

```yaml
document_type: api
target_audience: developer
content_source: |
  订单服务 API：
  - POST /orders - 创建订单
  - GET /orders/{id} - 获取订单
  - DELETE /orders/{id} - 取消订单
```

### 预期输出

- [ ] 包含 YAML 元数据头
- [ ] 包含 API 概述
- [ ] 每个接口有完整说明
- [ ] 包含请求/响应示例
- [ ] 结构规范、无渐进式披露

---

## 测试 2：人类模式 - 创建用户手册（执行型）

### 输入

```yaml
document_type: user-guide
target_audience: user
progressive_mode: execution
content_source: |
  系统登录功能：
  1. 用户访问系统网址
  2. 输入用户名和密码
  3. 点击登录按钮
```

### 预期输出

- [ ] 包含标题和一句话摘要（≤30字）
- [ ] 包含"快速了解"章节（3-5 条要点）
- [ ] 包含详细步骤说明
- [ ] 深度内容使用 `<details>` 折叠
- [ ] 包含 FAQ 章节
- [ ] 包含版本和日期元数据

## 测试 7：可读性验证

### 人类模式可读性

| 指标 | 目标值 |
|------|--------|
| 平均句长 | ≤ 25 字 |
| 最长句子 | ≤ 40 字 |
| 平均段长 | ≤ 5 句 |
| 标题深度 | ≤ 4 级 |

### 技术模式可读性

| 指标 | 目标值 |
|------|--------|
| 结构完整 | 元数据 + 概述 + 详情 |
| 示例代码 | 必须有注释 |
