# sop-dependency-manager 功能测试

## 测试 1: 安全漏洞依赖升级

### 前置条件
- 发现依赖有安全漏洞
- 有修复版本可用

### 输入
```
依赖名称：lodash
当前版本：4.17.20
目标版本：4.17.21
升级原因：安全漏洞修复
```

### 预期行为
1. 识别为 Patch 版本升级
2. 标记为 P0 级别
3. 检查 Breaking Changes（无）
4. 执行升级
5. 运行测试验证
6. 生成升级报告

### 验证点
- [ ] 是否识别为 P0
- [ ] 是否检查 Breaking Changes
- [ ] 是否运行测试
- [ ] 是否生成报告
