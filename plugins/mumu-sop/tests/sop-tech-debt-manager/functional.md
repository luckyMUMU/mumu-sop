# sop-tech-debt-manager 功能测试

## 测试 1: 债务识别与分类

### 前置条件
- 代码库可访问
- 有静态分析工具可用

### 输入
```
代码库目录：src/
关注领域：全部
```

### 预期行为
1. 运行静态分析识别代码坏味道
2. 分类债务类型（架构/代码/测试/文档/依赖）
3. 生成债务清单

### 验证点
- [ ] 是否识别了多种类型债务
- [ ] 是否正确分类
- [ ] 清单是否完整

## 测试 2: 债务评分与优先级

### 输入
```
债务项：支付模块缺少单元测试
```

### 预期行为
1. 评估业务影响（支付是核心功能 → 4分）
2. 评估偿还成本（约3天 → 3分）
3. 评估不偿风险（发布信心不足 → 4分）
4. 评估紧急程度（下次发布需要 → 4分）
5. 计算综合评分：3.8
6. 确定优先级：P1

### 验证点
- [ ] 评分是否基于客观标准
- [ ] 公式计算是否正确
- [ ] 优先级是否合理
