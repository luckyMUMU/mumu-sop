# sop-architecture-design 功能测试

## 测试 1: 新系统架构设计

### 输入
```
设计一个电商系统，包含：
- 用户管理
- 商品管理
- 订单管理
- 支付集成
需要支持高并发，预计日活用户 10 万。
```

### 预期输出
- 生成架构设计文档
- 定义模块边界和职责
- 定义模块间接口
- 评估技术选型

### 验证点
- [ ] 是否定义了清晰的模块边界
- [ ] 是否考虑了高并发需求
- [ ] 是否定义了接口契约
- [ ] 是否符合 P1 级规范

## 测试 2: 系统重构设计

### 输入
```
现有单体应用需要拆分为微服务架构。

```

### 预期行为

### 验证点
