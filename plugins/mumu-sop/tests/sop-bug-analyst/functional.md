# sop-bug-analysis 功能测试

## 测试 1: P0级安全漏洞分析

### 前置条件
- 用户报告了安全问题
- 可复现的安全漏洞

### 输入
```
Bug描述：用户可以通过修改URL参数查看其他用户的订单信息
复现步骤：
1. 登录用户A
2. 访问 /orders/123（属于用户B）
3. 成功看到用户B的订单
```

### 预期行为
1. 识别严重程度为 P0
2. 编写复现测试验证漏洞存在
3. 定位根因：权限校验缺失
4. 评估影响范围：所有订单数据
5. 生成包含安全修复建议的报告

### 验证点
- [ ] 是否识别为 P0 级别
- [ ] 是否创建了复现测试
- [ ] 是否定位到具体代码位置
- [ ] 是否评估了安全影响

## 测试 4: 完整分析流程

### 输入
```
用户反馈无法取消已发货的订单，点击取消按钮后提示"操作失败"。
```

### 预期行为（按顺序）
1. **信息收集**: 收集现象、复现步骤、影响范围
2. **复现测试**: 编写失败测试验证问题
3. **根因定位**: 分析代码找到根本原因
4. **影响评估**: 评估用户、功能、数据影响
5. **报告生成**: 生成完整分析报告

### 验证点
- [ ] 是否遵循完整流程
- [ ] 测试是否确实失败
- [ ] 报告是否包含所有必要字段
