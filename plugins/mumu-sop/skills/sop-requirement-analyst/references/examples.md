# sop-requirement-analyst 示例

## 输入示例

```
需求描述：
用户需要一个订单管理系统，支持创建订单、查询订单状态、取消订单。
订单需要记录商品信息、数量、总价。
用户只能取消未发货的订单。
```

## 输出示例

```markdown
# 订单管理规范

## 功能描述

订单管理系统支持创建、查询和取消订单。

## 业务规则

1. 订单必须包含至少一个商品
2. 订单总价 = 商品单价 × 数量
3. 只有未发货的订单可以取消

## 验收标准

### 创建订单

- 用户提交订单时，系统验证商品信息完整性
- 系统计算并记录订单总价
- 订单初始状态为"待支付"

### 取消订单

- 未发货订单可取消，状态变更为"已取消"
- 已发货订单不可取消，系统提示"已发货订单无法取消"
```

## 澄清记录格式

```markdown
# 需求澄清记录

## 第一轮：业务背景与目标

- 业务背景：解决什么问题？
- 目标用户：谁使用？

## 第二轮：长期目标与短期实现

- 长期愿景：最终状态
- 短期目标：MVP 范围
- 演进路径：迭代计划

## 第三轮：边界与约束

- 功能边界：不包含什么
- 技术约束：技术栈限制
- 资源限制：时间预算

## 第四轮：风险与假设

- 风险识别：最大风险
- 关键假设：已做的假设
- 回滚策略：应急预案
```
