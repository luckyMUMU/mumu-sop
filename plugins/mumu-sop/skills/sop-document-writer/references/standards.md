# 行业规范参考

本文档列出文档编写常用的行业规范和标准。

---

## 文档编写标准

### ISO 26514:2008 - 软件用户文档设计

**适用范围**：软件用户手册、操作指南

**核心要求**：
| 要素 | 要求 |
|------|------|
| 结构 | 清晰的层次结构，易于导航 |
| 语言 | 简洁、准确、一致 |
| 格式 | 统一的版式和排版 |
| 内容 | 完整覆盖用户任务 |

---

### IEEE 830-1998 - 软件需求规格说明

**适用范围**：需求文档编写

**SRS 结构模板**：
1. 引言（目的、范围、定义）
2. 总体描述（产品视角、功能、用户特征）
3. 具体需求（功能、性能、设计约束）

---

## 质量标准

### ISO 9001:2015 - 质量管理体系

**文档相关要求**：
- 文件控制程序
- 记录控制程序
- 版本控制
- 审批记录

---

## 可访问性标准

### WCAG 2.1 - Web 内容可访问性指南

**原则（POUR）**：
1. **可感知**：信息必须可被感知
2. **可操作**：界面组件必须可操作
3. **可理解**：信息和操作必须可理解
4. **健壮**：内容必须兼容各种工具

**文档具体要求**：
| 要求 | 说明 |
|------|------|
| 图片替代文本 | 所有图片提供 alt 文本 |
| 链接描述 | 链接文本描述目标 |
| 颜色对比 | 文字与背景对比度 ≥ 4.5:1 |

## 技术写作规范

### Microsoft Manual of Style

**核心原则**：
1. 使用简洁、直接的语言
2. 面向用户，不是面向系统
3. 使用主动语态
4. 避免行话和俚语

---

### Google Developer Documentation Style Guide

**结构原则**：
1. 先概述，后细节
2. 使用列表组织信息
3. 使用表格展示对比
4. 代码示例带注释

---

## 渐进式披露

### Nielsen Norman Group 研究发现

**核心原则**：
1. **降低认知负担**：初始视图只显示必要信息
2. **按需展开**：用户需要时提供更多信息
3. **可预测性**：用户知道在哪里找到更多信息
4. **可逆性**：用户可以收起不需要的信息
