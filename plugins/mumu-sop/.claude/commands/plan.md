启动 SOP 计划阶段，将规范分解为小而可验证的任务。

## 执行流程

1. **读取规范**
   - 加载 SPEC.md 或现有规范
   - 理解需求和约束

2. **垂直切片分解** (核心)
   - ❌ 避免水平切片：先全部数据库，再全部API，再全部UI
   - ✅ 采用垂直切片：每个任务交付完整功能路径
   
   示例：
   ```
   ❌ 水平切片（不推荐）
   Task 1: 构建整个数据库模式
   Task 2: 构建所有API端点
   Task 3: 构建所有UI组件
   
   ✅ 垂直切片（推荐）
   Task 1: 用户可以创建账户（模式+API+UI注册）
   Task 2: 用户可以登录（认证模式+API+UI登录）
   Task 3: 用户可以创建任务（任务模式+API+UI创建）
   ```

3. **创建任务依赖图**
   - 识别任务间的依赖关系
   - 确定并行执行的可能性

4. **生成任务列表**
   - 每个任务包含：描述、验收标准、验证方式、依赖项
   - 保存到 `.sop/specs/{change-id}/tasks.md`

## 关键原则

- 每个任务足够小，可在单次专注会话中完成
- 每个任务有明确的验收标准
- 优先构建基础依赖，再构建上层功能
- 任务大小：约 100 行代码变更

## 输出

- `tasks.md` - 任务分解文档
- `checklist.md` - 验证清单
