启动 SOP 构建阶段，通过增量实现交付功能。

## 执行流程

1. **读取任务列表**
   - 加载 plan 阶段生成的 tasks.md
   - 识别可并行的任务

2. **单切片实现** (核心)
   - 每次只实现一个垂直切片
   - 包含：数据层 → API层 → UI层
   - 每个切片：实现 → 测试 → 验证 → 提交

3. **P3→P0 约束检查**
   - 实现时遵循 P3 实现规范
   - 向上验证 P2、P1、P0 约束
   - 任何 P0 违反立即熔断

4. **增量提交**
   - 每个切片完成后提交
   - 提交信息遵循 Conventional Commits
   - 保持构建始终可用

## 实现原则

### 垂直切片
```
不要：等待所有层都完成
要：先完成一个端到端功能
```

### 安全默认
- 使用功能开关保护新功能
- 变更可回滚
- 不破坏现有功能

### 三击熔断
- 连续3次失败后停止执行
- 通知用户并记录问题
- 转入调试阶段

## 输出

- 代码变更
- 单元测试
- 提交历史
