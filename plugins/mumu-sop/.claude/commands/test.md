启动 SOP 测试阶段，证明代码按预期工作。

## 执行流程

1. **测试金字塔执行**
   - 单元测试（80%）- 快速反馈
   - 集成测试（15%）- 模块交互
   - E2E测试（5%）- 端到端场景

2. **覆盖率验证**
   - 核心模块必须 100% 覆盖
   - 整体覆盖率达到项目标准
   - 检查未覆盖代码的合理性

3. **Bug 分析与修复**
   - 测试失败时分析根因
   - 创建最小复现测试
   - 修复后验证回归

4. **浏览器测试**（前端项目）
   - DOM 结构验证
   - 控制台错误检查
   - 网络请求验证
   - Core Web Vitals 测量

## 测试原则

### DAMP 而非 DRY
- 测试代码可读性优先于复用
- 每个测试自包含，不依赖外部状态
- 测试名称描述行为，而非实现

### Beyoncé 规则
> "If you liked it, you should have put a test on it"
- 发现 Bug → 先写测试复现 → 再修复
- 新功能必须有测试覆盖

### Red-Green-Refactor
1. Red: 写测试，看到失败
2. Green: 写最小代码让测试通过
3. Refactor: 优化代码，保持测试通过

## 输出

- 测试报告
- Bug 修复记录
- 覆盖率报告
