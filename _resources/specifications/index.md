# 规范索引

> **规范层级**: P1-P2 级（系统/模块规范）

---

## 概述

本目录存放系统级和模块级规范，是 SOP 规范体系的核心组成部分。

---

## 规范层级说明

### P0 级：工程宪章

**位置**: `_resources/constitution/`

**特征**:
- 不可违背，违反即熔断
- 定义项目愿景、质量红线、架构原则、安全基线
- 变更需技术委员会审批

**文档**:
- [架构原则](../constitution/architecture-principles.md)

### P1 级：系统规范

**位置**: `_resources/specifications/`

**特征**:
- 跨模块约束
- 定义系统功能边界、核心业务流程
- 变更需技术负责人审批

**文档**:
- 系统规范（本目录）

### P2 级：模块规范

**位置**: `_resources/specifications/`

**特征**:
- 单模块约束
- 定义 API 契约、数据模型、领域模型
- 变更需模块负责人审批

**文档**:
- API 契约
- 数据模型
- 领域模型

### P3 级：实现规范

**位置**: `_resources/templates/`

**特征**:
- 实现细节
- 编码规范、测试规范、文档规范
- 自动化工具验证

## 规范创建指南

### 创建时机

| 规范层级 | 创建时机 | 创建者 |
|----------|----------|--------|
| P0 级 | 项目启动时 | 技术委员会 |
| P1 级 | 0→1 核心功能开发时 | 架构师 |
| P2 级 | 有明确需求时 | 模块负责人 |
| P3 级 | 实现阶段 | 开发者 |

### 创建流程

```
需求分析
    ↓
确定规范层级
    ↓
选择规范模板
    ↓
编写规范内容
    ↓
规范审查
    ↓
审批通过
    ↓
发布并归档
```

### 规范模板

#### 系统规范模板

```markdown
# [系统名称] 系统规范

## 功能边界
[定义系统功能范围]

## 核心业务流程
[描述核心业务流程]

## 外部接口定义
[定义与外部系统的接口]

## 性能指标
[定义性能要求]
```

#### API 契约模板

```yaml
openapi: 3.0.0
info:
  title: [API 名称]
  version: 1.0.0
paths:
  /api/resource:
    get:
      summary: [接口描述]
      responses:
        '200':
          description: 成功
```

## 规范演化规则

### 演化流程

```
轻规范（提案）
    ↓ 确认
技术确认
    ↓ 实现
代码实现
    ↓ 验证
测试通过
    ↓ 归档
升级为重规范
```

### 升级条件

| 条件 ID | 条件描述 | 升级目标 |
|---------|---------|---------|
| UPG-001 | 影响核心架构原则 | P0 级 |
| UPG-002 | 新增安全要求或发现安全漏洞 | P0 级 |
| UPG-003 | 新增核心业务流程 | P1 级 |
| UPG-004 | 新增/修改核心 API | P2 级 |
| UPG-005 | 新增/修改核心数据模型 | P2 级 |
| UPG-006 | 新增核心领域概念 | P2 级 |

### 版本管理

- 所有规范纳入 Git 管理
- 版本号遵循 SemVer（主版本.次版本.修订号）
- 每次变更必须有变更日志（CHANGELOG）
